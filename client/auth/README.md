# client/auth

This package is hand-written (not generated) and provides Omada-specific authorization helpers used by the generated SDK client.

Why it exists:

- The upstream OpenAPI spec does not currently model the Omada auth flow in a way that is convenient for direct generated usage.
- This package keeps auth logic explicit, small, and testable.

What it provides:

- Login/code/token helpers for Omada auth endpoints under `/openapi/authorize/*`.
- `AccessTokenRequestEditor` to set `Authorization: AccessToken=<token>`.
- Minimal request/response types used by auth calls.

How it is used:

- The top-level client wrappers in `client/omada_auth.go` delegate to this package.
- Most consumers should call the wrappers on `client.Client` / `client.ClientWithResponses`, not this package directly.

Notes:

- `client/client.go` remains generated by oapi-codegen.
- This package is intentionally maintained manually.
